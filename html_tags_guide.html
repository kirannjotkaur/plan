01

<html> — Root Element

The container that wraps everything

The <html> element is the root (top-level) element of an HTML document. Every other element must be a descendant of this element. Think of it as the outer wall of a house — everything lives inside it.

🌲 Document Tree

<html lang="en"> ← Root of everything
│ <head></head> ← Metadata
│ <body></body> ← Visible content
</html>

Code Example

<!DOCTYPE html>
<html lang="en">
  <head>...</head>
  <body>...</body>
</html>
AttributeDescription
langDeclares the language of the document (e.g. "en" for English)
dirText direction: ltr (left-to-right) or rtl (right-to-left)
💡 Always include lang="en" (or your language). Screen readers and search engines use it.
02

<head> — Metadata Container

The brain behind the page

The <head> element contains machine-readable information (metadata) about the document. Content inside <head> is never displayed directly on the page.

🗂️ What Goes in <head>

Code Example

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width">
  <title>My Page</title>
  <link rel="stylesheet" href="style.css">
</head>
💡 The <head> is invisible to users but critical for browsers, SEO crawlers, and social media link previews.
03

<title> — Page Title

What the browser tab shows

The <title> tag sets the title of the HTML page. It appears in the browser tab, bookmarks, and search engine results. It must be placed inside the <head> element.

🖥️ Browser Tab Diagram

Code Example

<head>
  <title>My Awesome Website</title>
</head>
💡 Keep titles under 60 characters for SEO. Google truncates longer titles in search results.
04

<body> — Document Body

Everything you see on the page

The <body> element contains all the content that is displayed in the browser window — text, images, links, buttons, forms, everything visual.

📄 Page Structure

Code Example

<body>
  <h1>Hello World</h1>
  <p>Welcome to my website.</p>
</body>

05

<h1> — Top-Level Heading

The biggest, boldest title on the page

HTML provides 6 levels of headings: <h1> through <h6>. <h1> is the most important and should appear once per page as the main topic. Each level gets progressively smaller.

📐 Heading Hierarchy

Code Example

<h1>Main Title</h1>
<h2>Section Heading</h2>
<h3>Sub-section</h3>
Live Preview

Main Title

Section Heading

Sub-section

💡 Use only one <h1> per page for good SEO structure. Use headings in order — don't skip from h1 to h4.
06

<p> — Paragraph

Block of readable text

The <p> tag defines a paragraph. Browsers automatically add space (margin) before and after each paragraph. It is a block-level element.

Code Example

<p>This is the first paragraph of text.</p>
<p>This is the second paragraph. Notice
the gap between them.</p>
Live Preview

This is the first paragraph of text.

This is the second paragraph. Notice the gap between them.

💡 Never use empty <p></p> tags to add spacing — use CSS margin instead.
07

<a> — Anchor / Hyperlink

The web's connecting glue

The <a> (anchor) element creates a hyperlink. The href attribute specifies the URL destination. Links can point to pages, sections, emails, phone numbers, or files.

🔗 Link Anatomy

Code Example

<!-- External link -->
<a href="https://google.com" target="_blank">
  Visit Google
</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Us</a>
Live Preview

Visit Google  |  Email Us

AttributeDescription
hrefURL the link points to
target="_blank"Opens link in a new tab
rel="noopener"Security for _blank links
08

<img> — Image

Embed pictures into your page

The <img> tag embeds an image. It is a self-closing (void) element — no closing tag needed. The src attribute specifies the image source, and alt provides alternative text for accessibility.

🖼️ Image Tag Anatomy

Code Example

<img
  src="https://placecats.com/200/150"
  alt="A cute cat"
  width="200"
  height="150"
>
Live Preview
💡 Always write meaningful alt text. Screen readers announce it, and it shows if the image fails to load.
09

<ul> — Unordered List

Bullet-point lists

The <ul> element creates a bulleted (unordered) list. Each item is wrapped in a <li> tag. Use when order doesn't matter.

Code Example

<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Cherries</li>
</ul>
Live Preview
  • Apples
  • Bananas
  • Cherries
10

<ol> — Ordered List

Numbered, step-by-step lists

The <ol> element creates a numbered list. Use when sequence matters — steps, rankings, recipes. The browser automatically numbers the <li> items.

Code Example

<ol>
  <li>Boil water</li>
  <li>Add pasta</li>
  <li>Cook 10 minutes</li>
</ol>
Live Preview
  1. Boil water
  2. Add pasta
  3. Cook 10 minutes
💡 Use type="A" for letters or type="i" for Roman numerals. Use start="5" to begin numbering from 5.
11

<li> — List Item

Each item inside a list

The <li> element represents a single item in a list. It must always be inside a <ul>, <ol>, or <menu> element. It can contain text, images, links, or even nested lists.

<ul>

├── <li>Item 2</li>
└── <li>Item 3 (with nested list)
<ul>

</li>
</ul>

<div> — Division / Container

The universal layout box

The <div> is a generic block-level container with no semantic meaning. It groups elements for styling with CSS or manipulation with JavaScript. Think of it as an invisible box.

📦 Div as Layout Boxes

Code Example

<div class="card">
  <h2>Card Title</h2>
  <p>Card content goes here.</p>
</div>
Live Preview

Card Title

Card content goes here.

13

<span> — Inline Container

Style part of a line of text

The <span> is an inline container with no default styling or meaning. It is used to apply CSS or JavaScript to a specific part of text without breaking the flow of the line.

Code Example

<p>
  The price is <span style="color:red; font-weight:bold">
    $9.99
  </span> only today!
</p>
Live Preview

The price is $9.99 only today!

💡 <div> is block-level (starts a new line). <span> is inline (stays within text flow).
14

<table> — Table

Rows and columns of data

The <table> element creates a data table. It contains rows (<tr>), which contain header cells (<th>) or data cells (<td>). Tables should be used for tabular data, not page layout.

🗃️ Table Structure

Code Example

<table>
  <tr>
    <th>Name</th>  <th>Age</th>  <th>City</th>
  </tr>
  <tr>
    <td>Alice</td> <td>24</td>   <td>Delhi</td>
  </tr>
  <tr>
    <td>Bob</td>   <td>31</td>   <td>Mumbai</td>
  </tr>
</table>
Live Preview

Alice24Delhi
Bob31Mumbai
15

<tr> — Table Row

A single horizontal row in a table

The <tr> element defines a row in an HTML table. Each <tr> contains one or more <th> or <td> cells. Rows are stacked top to bottom.

💡 <tr> must always be a direct child of <table>, <thead>, <tbody>, or <tfoot>.
16

<td> — Table Data Cell

A single cell of data

The <td> element defines a data cell in a table. Use <th> for header cells (bold, centered by default). The colspan and rowspan attributes let cells span multiple columns or rows.

colspan / rowspan Example

<tr>
  <td colspan="2">Spans 2 columns</td>
</tr>
17

<form> — HTML Form

Collect input from users

The <form> element groups interactive controls that a user can fill in and submit. The action attribute specifies where to send the data; method specifies how (GET or POST).

📬 Form Data Flow

Code Example

<form action="/submit" method="post">
  <input type="text" placeholder="Your name">
  <button type="submit">Submit</button>
</form>
Live Preview
18

<input> — Input Field

The most versatile form element

The <input> element creates an interactive field for user data. It is a void element (no closing tag). The type attribute dramatically changes its behaviour and appearance.

🎛️ Input Types

Code Example

<input type="text"     placeholder="Name">
<input type="email"    placeholder="Email">
<input type="password" placeholder="Password">
<input type="checkbox"> Remember me
<input type="submit"   value="Send">
19

<button> — Clickable Button

Trigger actions on click

The <button> element creates a clickable button. Unlike <input type="submit">, it can contain HTML content like icons and styled text. The type attribute controls behaviour.

Code Example

<!-- Submit a form -->
<button type="submit">Submit Form</button>

<!-- JavaScript action -->
<button onclick="alert('Hello!')">
  Say Hello
</button>

<!-- Disabled button -->
<button disabled>Can't click me</button>
Live Preview

buttonNo default action — use with JavaScript

20

<nav> — Navigation

The semantic menu container

The <nav> element is a semantic HTML5 element that represents a section containing navigation links. It communicates to browsers and screen readers that this group of links is for navigating the site.

🧭 Nav in Page Layout

Code Example

<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>
  <a href="/contact">Contact</a>
</nav>
Live Preview
HomeAboutContact
💡 <nav> is semantic — it has the same visual effect as <div> but tells assistive technologies "these are navigation links."
